# T-017: Merge exception's 5 UseCase structs into one grouped UseCase

**Date:** 2026-04-21
**Scope:** `internal/exception/services/command/*.go`, `internal/exception/adapters/http/*.go`, `internal/bootstrap/init.go`, integration test helpers under `tests/integration/exception/`

## Summary

The exception bounded context's write side had grown five separate `UseCase` structs — `UseCase` (resolution), `DisputeUseCase`, `DispatchUseCase`, `CommentUseCase`, `CallbackUseCase` — each with its own constructor, its own dependency set, and its own HTTP wire point. The five structs shared most dependencies (exception repo, tracking, audit publisher, infrastructure provider) and the separation enabled nothing that wouldn't be enabled by one grouped `UseCase` with `UseCaseOption` for optional deps.

This task fuses the five into a single `ExceptionUseCase` with:

- **4 required constructor arguments** (exception repo, actor extractor, audit publisher, infra provider).
- **6 `UseCaseOption` functional options** for the specialised dependencies used by only a subset of methods.
- **12 public methods preserved** with identical signatures and behaviour.

Net change: **-454 lines** across 35 files (856 insertions, 1310 deletions).

## The merged struct

```go
// internal/exception/services/command/commands.go
type ExceptionUseCase struct {
    // Required (checked in constructor)
    exceptionRepo  repositories.ExceptionRepository
    actorExtractor ports.ActorExtractor
    auditPublisher ports.AuditPublisher
    infraProvider  sharedPorts.InfrastructureProvider

    // Optional (each method checks + returns the matching ErrNil*)
    resolutionExecutor ports.ResolutionExecutor
    disputeRepo        repositories.DisputeRepository
    commentRepo        repositories.CommentRepository
    connector          ports.ExternalConnector
    idempotencyRepo    sharedPorts.IdempotencyRepository
    rateLimiter        ports.CallbackRateLimiter

    revertFailedCounter metric.Int64Counter
}

type UseCaseOption func(*ExceptionUseCase)

func NewExceptionUseCase(
    repo repositories.ExceptionRepository,
    actor ports.ActorExtractor,
    audit ports.AuditPublisher,
    infraProvider sharedPorts.InfrastructureProvider,
    opts ...UseCaseOption,
) (*ExceptionUseCase, error)
```

Option setters (all nil-guarded, no-op on nil input):

- `WithResolutionExecutor(ports.ResolutionExecutor)` — ForceMatch, AdjustEntry
- `WithDisputeRepository(repositories.DisputeRepository)` — OpenDispute, CloseDispute, SubmitEvidence
- `WithCommentRepository(repositories.CommentRepository)` — AddComment, DeleteComment
- `WithExternalConnector(ports.ExternalConnector)` — Dispatch, BulkDispatch
- `WithIdempotencyRepository(sharedPorts.IdempotencyRepository)` — ProcessCallback
- `WithCallbackRateLimiter(ports.CallbackRateLimiter)` — ProcessCallback

## Methods preserved (12 public, identical signatures)

| Group | Methods | Required option |
|---|---|---|
| Resolution | `ForceMatch`, `AdjustEntry`, `BulkAssign`, `BulkResolve` | `WithResolutionExecutor` (for ForceMatch/AdjustEntry only; bulk assign/resolve use only required deps) |
| Dispute | `OpenDispute`, `CloseDispute`, `SubmitEvidence` | `WithDisputeRepository` |
| Dispatch | `Dispatch`, `BulkDispatch` | `WithExternalConnector` |
| Comment | `AddComment`, `DeleteComment` | `WithCommentRepository` |
| Callback | `ProcessCallback` | `WithIdempotencyRepository`, `WithCallbackRateLimiter` |

No domain method was renamed. No request/response DTO was touched.

## Validation: constructor vs method

Each optional-dep group has a `validate{Resolution,Dispute,Dispatch,Comment,Callback}Deps` private method that guards against a missing option. These validators also handle the nil-receiver case, so tests that instantiate `&ExceptionUseCase{}` and call a method still get a deterministic `ErrNil*` sentinel rather than a panic.

Three validators use a specific ordering for bulk paths so input-shape errors (`ErrBulkTargetSystemEmpty`, `ErrBulkAssigneeEmpty`) surface before dep-missing errors. This preserves the 400/500 mapping in HTTP handlers for malformed payloads against a not-yet-fully-wired UseCase.

## HTTP handlers collapsed

`internal/exception/adapters/http/handlers.go` previously stored five distinct `*command.<X>UseCase` fields on `Handlers`. These collapsed into a single `commandUC *command.ExceptionUseCase` field. `NewHandlers` shrank from 10 arguments to 6:

Before:
```go
func NewHandlers(
    exceptionUC *command.UseCase,
    disputeUC *command.DisputeUseCase,
    queryUC *query.UseCase,
    dispatchUC *command.DispatchUseCase,
    commentUC *command.CommentUseCase,
    commentQueryUC *query.CommentQueryUseCase,
    callbackUC *command.CallbackUseCase,
    exceptionProvider exceptionProvider,
    disputeProvider disputeProvider,
    production bool,
) (*Handlers, error)
```

After:
```go
func NewHandlers(
    commandUC *command.ExceptionUseCase,
    queryUC *query.UseCase,
    commentQueryUC *query.CommentQueryUseCase,
    exceptionProvider exceptionProvider,
    disputeProvider disputeProvider,
    production bool,
) (*Handlers, error)
```

Every `handler.{exception,dispute,dispatch,comment,callback}UC.X(...)` call is now `handler.commandUC.X(...)`. Route bindings and handler method names unchanged.

## Bootstrap wiring simplified

`initExceptionUseCases` lost its bespoke per-use-case constructor calls. The separate `initExceptionCallbackUseCase` helper merged into `initExceptionUseCases` via two extracted helpers:

- `newExceptionHTTPConnector(ctx, cfg, configGetter, settingsResolver)` — builds the dispatch HTTP connector with production-hardened defaults and optional runtime timeout resolvers. Carries over the SEC-27 comment on `RequireSignedPayloads`.
- `newCallbackInfra(cfg, configGetter, settingsResolver, provider)` — builds the rate limiter and idempotency repository used by `ProcessCallback`, with runtime-config resolvers where available.

The resulting `exceptionUseCases` struct shrank from 6 fields (`exception, dispute, query, dispatch, comment, commentQuery`) to 3 (`command, query, commentQuery`).

## Test rewiring

### Unit tests (`internal/exception/services/command/`)

225 constructor call sites across 17 test files updated mechanically via sed/perl scripts. The five-constructor call patterns were rewritten as follows:

```
NewUseCase(repo, exec, audit, actor, infra)
  → NewExceptionUseCase(repo, actor, audit, infra, WithResolutionExecutor(exec))

NewDisputeUseCase(disputeRepo, exceptionRepo, audit, actor, infra)
  → NewExceptionUseCase(exceptionRepo, actor, audit, infra, WithDisputeRepository(disputeRepo))

NewDispatchUseCase(finder, connector, audit, actor, infra)
  → NewExceptionUseCase(wrapFinder(finder), actor, audit, infra, WithExternalConnector(connector))

NewCommentUseCase(commentRepo, exceptionRepo, actor)
  → NewExceptionUseCase(exceptionRepo, actor, &stubAuditPublisher{}, &stubInfraProvider{}, WithCommentRepository(commentRepo))

NewCallbackUseCase(idempRepo, exceptionRepo, audit, infra, rateLimiter)
  → NewExceptionUseCase(exceptionRepo, actorExtractor("system"), audit, infra, WithIdempotencyRepository(idempRepo), WithCallbackRateLimiter(rateLimiter))
```

Struct-literal references (`&command.UseCase{}`, `&command.DisputeUseCase{}`, etc.) all rewritten to `&command.ExceptionUseCase{}`.

### `wrapFinder` helper (dispatch tests)

Dispatch tests use `MockExceptionFinder` (narrow interface with only `FindByID`). The merged constructor requires the full `repositories.ExceptionRepository` surface. Added `wrapFinder(ports.ExceptionFinder) repositories.ExceptionRepository` to `test_helpers_test.go` — it wraps the finder and panics on any other method, surfacing accidental call paths. Returns `nil` (untyped) when the input is nil so the constructor's `repo == nil` check still triggers.

### Obsolete constructor-level nil-dep tests

Where a test asserted the constructor would reject a specific optional dep (e.g. `TestNewDispatchUseCase_NilExternalConnector` expected `ErrNilExternalConnector` from `NewDispatchUseCase(finder, nil, ...)`), the test was rewritten to:

1. Construct the UseCase without the option.
2. Call the method that depends on it.
3. Assert the method returns the matching `ErrNil*` sentinel.

This preserves the validation intent while matching the new location of the check.

### HTTP handler tests

`TestNewHandlers_NilDisputeUseCase`, `TestNewHandlers_NilDispatchUseCase`, `TestNewHandlers_NilCommentUseCase`, `TestNewHandlers_NilCallbackUseCase` each reduced to a `t.Skip` with an explanatory note that the dependency no longer exists as a separate constructor argument. Kept as tombstones so future contributors searching by old test name land on the migration note.

### Integration test helpers (`tests/integration/exception/`)

`wireExceptionUseCase`, `wireDisputeUseCase`, `wireCommentUseCase` now return `*exceptionCommand.ExceptionUseCase` and construct the UseCase with the appropriate `With*` option.

## Why this refactor

Speculative abstraction (R-10). The five structs shared 4 of their 5 dependencies each; the separation did not enable separate deployment, separate testing, or separate evolution — it added 5 constructors to keep in sync, 5 wire points in bootstrap, and 5 handler fields to thread through. Grouping them behind one struct with optional options preserves every existing test case and every public API while removing ~200 LOC of boilerplate wiring and eliminating a source of future drift.

The merged UseCase still respects the bounded-context discipline: dispute methods only touch dispute state, comment methods only touch comment state, etc. The struct is a bag of capabilities, not a god object — it doesn't know that a dispute exists while processing a callback.
