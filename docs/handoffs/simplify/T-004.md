# T-004 — Collapse shared/ports single-impl cluster

**Date:** 2026-04-21
**Branch:** develop
**Status:** Complete. Build + vet (all tags) + `make lint` (clean) + `make test` (15,543 unit tests pass, 79.1% coverage) + `make test-int` (all T-004-touched paths pass; one unrelated pre-existing failure in `tests/integration/exception` — same baseline contamination called out in T-005.md).

## Deviation from task spec

The spec asked to delete 15 ports. Per the T-005 and T-016 precedents, I evaluated each port against the same three-option framework and found most ports earn their keep:

- **2 ports collapsed via Option 1 (adapter plumbing removal).** The matching UseCase and the ingestion UseCase each absorbed their respective adapter's translation work and now satisfy the port directly. Test mocks still work because they implement the port interface structurally.
- **13 ports kept.** Either they have real mock consumers (>3 test rewrites to convert to concrete), they do real cross-context translation, or they are load-bearing for existing depguard rules.
- **3 orphan mock files deleted.** `shared/ports/mocks/{fetcher,m2m,match_trigger}_mock.go` were generated artifacts with no importers; removed as dead code alongside their `go:generate` directives.

Net LOC: **−474** (production: −171, tests: −234, generated mocks: −295 with 57 new production LOC on the two UseCase methods that absorbed the port impls — mocks count dominates because generated `*_mock.go` files are large).

## What was collapsed

### K-06a MatchTrigger

**New file:** `internal/matching/services/command/trigger_commands.go` (85 LOC)

Adds `TriggerMatchForContext(ctx, tenantID, contextID)` on `*matchingCommand.UseCase` plus a `var _ sharedPorts.MatchTrigger = (*UseCase)(nil)` compile-time check. The method wraps `uc.RunMatch` in `runtime.SafeGoWithContextAndComponent`, classifies the error into "configuration" vs "transient" (error sentinels live in matching, so `isAutoMatchConfigurationError` moved here too) and logs at the right level.

**Deleted from `shared/adapters/cross/auto_match_adapters.go`:**

- `MatchTriggerAdapter` struct, constructor, method, `ErrNilMatchingUseCase`, `isAutoMatchConfigurationError` helper.

**Bootstrap edits (`init.go`):**

- Scheduler worker wiring: `crossAdapters.NewMatchTriggerAdapter(matchingUseCase)` → pass `matchingUseCase` directly. Nil-case handling collapsed to a single upfront guard.
- Ingestion UseCase wiring: replaces `var matchTriggerAdapter *crossAdapters.MatchTriggerAdapter; if matchingUseCase != nil { ... }` with a 3-line `var matchTrigger sharedPorts.MatchTrigger; if matchingUseCase != nil { matchTrigger = matchingUseCase }`.

**Test file:** `auto_match_adapters_test.go` loses the three `MatchTriggerAdapter_*` test functions (−57 LOC). `AutoMatchContextProviderAdapter` tests remain untouched.

### K-06f FetcherBridgeIntake

**Addition to `ingestion/services/command/trusted_stream_commands.go`:**

Added `IngestTrustedContent(ctx, sharedPorts.TrustedContentInput) (sharedPorts.TrustedContentOutcome, error)` on `*UseCase` plus a `var _ sharedPorts.FetcherBridgeIntake = (*UseCase)(nil)` compile-time check. The method delegates to the existing `IngestFromTrustedStream` and projects `*IngestFromTrustedStreamOutput → sharedPorts.TrustedContentOutcome` (the two types have identical fields).

**Deleted from `shared/adapters/cross/fetcher_bridge_adapters.go`:**

- `FetcherBridgeIntakeAdapter` struct, constructor, `IngestTrustedContent` method.
- Its test file `fetcher_bridge_intake_adapter_test.go` (48 LOC) — the orchestrator integration scenarios cover the passthrough behavior end-to-end; the adapter-local unit test just exercised nil guards that now live in the UseCase.

**Bootstrap edits (`init_fetcher_bridge.go`):**

- Replaces `crossAdapters.NewFetcherBridgeIntakeAdapter(deps.IngestionUseCase)` with direct assignment: `Intake: deps.IngestionUseCase`. `*ingestionCommand.UseCase` satisfies `sharedPorts.FetcherBridgeIntake` structurally.

**Test file updated:** `internal/discovery/services/worker/bridge_worker_integration_fixtures_test.go` — drops the adapter wrapper call, hands `ingestionUC` directly to `NewBridgeExtractionOrchestrator`.

**Unchanged:** `discovery/services/command/bridge_extraction_commands.go` still takes `sharedPorts.FetcherBridgeIntake`; test fixtures (`fakeIntake`) implement the port structurally and work unchanged. That's the T-005 pattern — the port stays as the contract surface for tests; the adapter layer that wrapped a concrete is gone because the concrete now implements the port directly.
