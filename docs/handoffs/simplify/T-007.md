## Divergence finding: ReconciliationSource

The task spec named this as a K-12 candidate, but per the hard constraint:

> "Before deletion, verify 'byte-identical' claim via diff of struct fields. If divergence exists (even whitespace or field-order), stop and document in handoff."

The two declarations differ in one field:

| Field | `internal/shared/domain/field_map.go â†’ ReconciliationSource` | `internal/configuration/domain/entities/reconciliation_source.go â†’ ReconciliationSource` |
|---|---|---|
| ID | `uuid.UUID` | `uuid.UUID` |
| ContextID | `uuid.UUID` | `uuid.UUID` |
| Name | `string` | `string` |
| **Type** | **`string`** | **`value_objects.SourceType`** |
| Side | `sharedfee.MatchingSide` | `sharedfee.MatchingSide` |
| Config | `map[string]any` | `map[string]any` |
| CreatedAt | `time.Time` | `time.Time` |
| UpdatedAt | `time.Time` | `time.Time` |

The shared kernel version uses raw `string` for `Type` to avoid pulling `internal/configuration/domain/value_objects` into `internal/shared/`. The configuration entity uses the typed enum `value_objects.SourceType` (with `Valid()`, `String()`, `ParseSourceType()` behavior). The `toSharedReconciliationSource` copy function does `Type: string(src.Type)` â€” it deliberately downcasts.

Unifying would require one of:

1. **Move `value_objects.SourceType` to `internal/shared/domain/` too.** This expands scope to 33 more files. Not in T-007 mandate.
2. **Relax the configuration entity's `Type` to `string`.** Regresses enum validation â€” a net loss in type safety.
3. **Keep the duplication and the cross-adapter copy.** Costs: 2 redundant type declarations + one copy function. Buys: enum safety inside configuration + zero-dep shared kernel.

Picked option 3. Flagged as future cleanup candidate; appropriate parent task would be a shared-kernel promotion of `SourceType` (companion to the MEMORY.md "severity pattern" for multi-kernel enums).

The shared `ReconciliationSource` docstring now points at this handoff for the rationale so future readers don't re-relitigate.
