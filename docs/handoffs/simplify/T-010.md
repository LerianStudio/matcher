# T-010: Delete type-alias shim files in domain/value_objects/

**Date:** 2026-04-21
**Scope:** 5 shim files + 6 redundant shim tests deleted; 119 caller files migrated to import `shared/domain` + `shared/domain/exception` directly.

## Summary

Deleted 5 type-alias shim files that re-exported `shared/domain` and `shared/domain/exception` symbols under `internal/exception/domain/value_objects/` and `internal/configuration/domain/value_objects/`. In a pre-public application with no external consumers, "backward-compat" aliases add exactly one thing: a second name for the same type. Callers now import the canonical definitions directly.

Also deleted 6 test files that only exercised the shim re-exports (the shared packages already ship full test coverage for the underlying types).

The idempotency/context/rule shims also wrapped `Parse*` functions with a redundant `fmt.Errorf("parsing …: %w", err)` layer. Extra wrap layer dropped along with the shims — effectively closes T-011 for these files.

## Migrated callers (119 files)

All references to shim symbols rewritten:

| Old alias / package | Shim symbol | New alias / package |
|---|---|---|
| `value_objects.ExceptionSeverity*` et al. | severity family (14 symbols) | `sharedexception.*` → `internal/shared/domain/exception` |
| `value_objects.IdempotencyKey` et al. | idempotency family (11 symbols) | `shared.*` → `internal/shared/domain` |
| `value_objects.ContextType*` et al. | context/rule families (12 symbols) | `shared.*` → `internal/shared/domain` |
| `exceptionVO.*` (same symbols) | — | `shared.*` / `sharedexception.*` |
| `configVO.*` (same symbols) | — | `shared.*` |

Migration applied via perl word-boundary substitution, then imports injected via `golang.org/x/tools/go/ast/astutil.AddNamedImport`. Stale aliased imports (`exceptionVO`, `configVO`, bare `value_objects`) that became unused were pruned explicitly — `goimports` skips files whose build tags it cannot satisfy, so direct removal was required for 12 `//go:build integration` files.

Callers that still reference non-shim symbols in the same package (e.g. `value_objects.ExceptionStatus`, `value_objects.CurrencyCode`, `value_objects.ContextStatus`, `value_objects.SourceType`) kept the original `value_objects` import intact alongside the new `shared` / `sharedexception` imports. Typical caller now has two imports instead of one — declared cost of making the canonical home visible.
