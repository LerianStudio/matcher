# T-008: Delete pkg/errors.go typed wrappers

**Date:** 2026-04-21
**Scope:** `pkg/errors.go`, `pkg/errors_test.go`, 3 test files under `internal/shared/adapters/http/`

## Summary

Deleted 10 typed error wrapper structs and their 10 constructors from `pkg/errors.go`. The K-17 simplify audit verified zero production code dispatched on these concrete types via `errors.As`. The wrappers existed as scaffolding that validated its own existence through a dedicated test (`TestNewError_SelectsTypedErrorByStatus`) without affecting runtime behavior. Deleting them collapses the error model to a single concrete type (`*BaseError`) while preserving the full `APIError` interface contract.

The 4 `typedNil` references were there to test nil-safety of dispatch through `APIError` — legitimate coverage of `isNilError`, `asProductError`, `RespondProductError`, `ValidateFallbackError`. Any typed-nil pointer-to-APIError-implementer suffices; `*BaseError` replaces `*NotFoundError` with identical semantics (both satisfy `APIError`, both are caught by the reflection-based nil check in `isNilError`).

## Why `*BaseError` (not raw interface nil)

The tests deliberately exercise the "typed-nil behind interface" path — a notorious Go gotcha where `err != nil` returns true even though the concrete value is nil. The `isNilError` helper uses reflection to detect this. Replacing `*NotFoundError` (now deleted) with `*BaseError` preserves the test's intent: a non-nil interface wrapping a nil pointer whose concrete type implements `APIError`.
