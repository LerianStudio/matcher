## Open question resolution (Fix #4 tension)

**Question raised:** Does narrowing carve-outs #1 and #2 to exclude
`domain/repositories/` break what T-004 and T-005 actually need to import?

**Answer:** No. Resolved by reading `docs/pre-dev/simplify-2026-04-21/tasks.md`
lines 280-417:

- **T-004 (line 286):** Deletes `internal/shared/ports/` single-impl interfaces;
  "consumers reference concrete types directly." T-004's configuration
  consumers (e.g. post-K-06b shim deletion) import configuration entities /
  value_objects, NOT `configuration/domain/repositories`. Fix #4's deny on
  `configuration/domain/repositories` does not block T-004.
- **T-005 (line 362):** Deletes matching-ports; "Matching UseCase takes
  concrete `*Repository` references." These concrete repos live at
  `ingestion/adapters/postgres/transaction/...`, not
  `ingestion/domain/repositories`. Importing concrete adapter types from
  matching's services is blocked by `service-no-adapters`, not by this rule —
  so T-005 will either import from matching's own adapter/ports composition,
  or (more likely) move the `TransactionRepository` interface to
  `shared/domain/` where no depguard rule denies it. Fix #4's deny on
  `ingestion/domain/repositories` does not block T-005 either.

Both task specs confirm: carve-outs #1 and #2 are for cross-context
**domain data types** only (entities + value_objects). Repo-interface
cross-context imports would be port coupling and are not part of the intent.

### Special note on probe 9 (entity-purity glob repair)

Initial probe passed when it was supposed to fail. Investigation revealed
the `entity-purity` rule's `files:` glob `**/domain/entities/**/*.go` had a
dormant bug: it requires at least one directory segment after `entities/`,
which no entity file in the repo currently has. The rule was never firing.
Fix: added `**/domain/entities/*.go` alongside the existing glob. With the
glob fixed, Fix #3's adapter denies now apply. Confirmed no pre-existing
entity files violate the rule (codebase was compliant by convention, not
enforcement) — baseline lint stayed clean after the glob fix. The glob
repair is an in-scope companion to Fix #3: without it, Fix #3 is cosmetic.
