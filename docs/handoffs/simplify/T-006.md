# T-006: Delete adapter-mocking interfaces

## Summary

Deleted four single-use interfaces in `internal/shared/adapters/cross/` that existed purely to enable unit-test mocking of adapter glue, plus five non-`WithTx` pass-through methods on `TransactionRepositoryAdapter`. Real behaviour is already covered by the ~55 integration tests added 2026-02-06 (MEMORY.md). Consumers now depend on concrete types; the matching-port `TransactionRepository` interface now exposes only `*WithTx` variants, aligning with the custom `repositorytx` linter's intent.

## Risk R-T006-2 assessment

The simplify task flagged the possibility that deleting non-`WithTx` pass-throughs might expose callers that *should* have been inside a transaction. Audit result:

- **Matching production code** (`internal/matching/services/command/**`) uses only `*WithTx` variants. No exposure.
- **`ExceptionMatchingGateway.CreateForceMatch`** calls `MarkMatched` on the gateway's `transactionRepo` field. That field is now the concrete `*ingestionTxRepo.Repository`, which retains its own non-`WithTx` `MarkMatched` (real work: wraps its own tenant-scoped transaction). This call is semantically correct — force-match targets a single transaction row; composing it with other writes inside a single tx is not required. **No fix needed.**
- **Integration-test callers** (`tests/integration/matching/cross_context_transactions_test.go`) were updated to use `adapter.WithTx(...)` — the new equivalent.
